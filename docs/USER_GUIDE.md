# 블로그 글 생성기 - 사용자 가이드

사진과 간단한 메모만 입력하면 AI가 네이버 블로그 스타일의 글을 자동으로 작성해 줍니다.

---

## 시작하기

### 1. 환경 준비

**필요 사항:**
- Java 17 이상
- Anthropic API 키 ([console.anthropic.com](https://console.anthropic.com)에서 발급)

### 2. API 키 설정

프로젝트 루트에 `.env` 파일을 생성하고 API 키를 입력합니다.

```bash
cp .env.example .env
```

`.env` 파일을 열어 실제 키를 입력:
```
CLAUDE_API_KEY=sk-ant-your-actual-key-here
```

### 3. 실행

```bash
./gradlew bootRun
```

> macOS에서 Java 버전 문제가 발생하면:
> ```bash
> JAVA_HOME=$(/usr/libexec/java_home -v 17) ./gradlew bootRun
> ```

브라우저에서 **http://localhost:8080** 접속.

## 블로그 글 생성하기

### Step 1. 사진 업로드

- **드래그 앤 드롭**: 사진 파일을 업로드 영역에 끌어다 놓기
- **클릭 선택**: 업로드 영역을 클릭해서 파일 선택

| 항목 | 조건 |
|---|---|
| 장수 | 1~10장 |
| 포맷 | JPEG, PNG, GIF, WebP |
| 용량 | 개별 파일 최대 10MB |

업로드하면 미리보기 그리드에 번호와 함께 썸네일이 표시됩니다.
이 번호가 생성된 글의 `[사진1]`, `[사진2]` 순서와 일치합니다.

> **Tip**: 사진에 GPS/촬영일 정보(EXIF)가 있으면 AI가 자동으로 활용합니다.
> 스마트폰으로 촬영한 사진은 대부분 EXIF가 포함되어 있어 장소/날짜가 글에 반영됩니다.

### Step 2. 메모 작성

AI에게 전달할 핵심 정보를 자유롭게 적어주세요. 길게 쓸 필요 없이 키워드 위주로 충분합니다.

**좋은 메모 예시:**
```
성수동 크림파스타 맛집
사진1: 외관 - 골목 안쪽에 있음
사진2: 크림파스타 - 진짜 맛있었음, 2만원
사진3: 디저트 티라미수 - 서비스로 줌
```

**메모에 포함하면 좋은 것들:**
- 장소 이름, 위치
- 메뉴명, 가격
- 인상 깊었던 점
- 사진별 간단한 설명

### Step 3. 글 스타일 선택

| 스타일 | 특징 | 추천 상황 |
|---|---|---|
| **친근한 일상** | "~했어요", "~인데요" 같은 구어체. 가볍고 친근한 톤 | 일상 기록, 맛집 방문, 카페 탐방 |
| **정보성 리뷰** | 위치, 가격, 메뉴를 체계적으로 정리. 실용적인 정보 전달 | 맛집 리뷰, 제품 리뷰, 장소 소개 |
| **여행 에세이** | 감성적이고 서정적인 문체. 느낌과 감동 중심 | 여행기, 풍경 사진, 특별한 경험 |

### Step 4. 생성 조건 설정 (선택)

AI에게 추가 지시사항을 전달할 수 있습니다. 비워두면 기본 규칙으로 생성됩니다.

**활용 예시:**
| 입력 | 효과 |
|---|---|
| `반말로 작성해 주세요` | "~했다", "~이다" 체로 변경 |
| `해시태그 5개 포함` | 글 끝에 `#성수동맛집` 같은 해시태그 추가 |
| `이모지 많이 사용` | 이모지를 더 풍부하게 배치 |
| `가격 정보를 꼭 포함하고 별점 매겨줘` | 가격표 + 별점 평가 포함 |
| `문장은 짧고 간결하게` | 짧은 문장 위주로 작성 |
| `~했다 체 사용` | 문어체로 변경 |

여러 조건을 `/`로 구분해서 한 번에 입력할 수도 있습니다:
```
반말 사용 / 이모지 많이 / 해시태그 5개 포함
```

### Step 5. 글 생성

**"글 생성하기"** 버튼을 클릭하면 AI가 글을 작성합니다.
보통 **30초~1분** 정도 소요됩니다.

## 생성 결과 활용하기

### 글 확인

생성이 완료되면 결과 화면에 다음이 표시됩니다:
- **제목**: SEO 키워드가 포함된 블로그 제목
- **본문**: `[사진N 삽입 위치]` 표시와 함께 완성된 글
- **사진 삽입 가이드**: 어떤 사진을 어디에 넣을지 안내

### 글 복사

| 버튼 | 용도 |
|---|---|
| **HTML 복사** | 네이버 블로그 에디터에 HTML 모드로 붙여넣기할 때 |
| **텍스트 복사** | 일반 텍스트로 복사해서 원하는 곳에 활용할 때 |

### 글 수정

생성된 글을 바로 수정할 수 있습니다. 하단의 **"수정 요청"** 입력란에 지시사항을 입력하고 **수정** 버튼을 클릭하세요 (또는 `Enter` 키).

**수정 요청 예시:**
- `제목 좀 더 짧게 바꿔줘`
- `마지막 문단 삭제해줘`
- `좀 더 감성적인 느낌으로`
- `가격 정보를 표로 정리해줘`
- `이모지를 좀 줄여줘`

수정은 여러 번 반복할 수 있으며, 이전 대화 맥락을 기억하므로 점진적으로 원하는 결과를 만들어 갈 수 있습니다.

### 새 글 작성

하단의 **"← 새 글 작성하기"** 버튼을 클릭하면 입력 화면으로 돌아갑니다.
이전에 입력한 메모, 스타일, 사진 미리보기가 그대로 남아 있어 바로 재생성할 수 있습니다.

---

## 네이버 블로그에 게시하기

1. **텍스트 복사** 클릭
2. 네이버 블로그 에디터에 붙여넣기
3. 글에서 `[사진1]`, `[사진2]` 등의 표시를 찾아 해당 위치에 사진 삽입
4. **사진 삽입 가이드**의 캡션을 참고하여 사진 설명(대체 텍스트) 입력
5. 최종 검토 후 발행

> **Tip**: HTML 복사를 사용하면 네이버 블로그의 HTML 편집 모드에서 서식(줄바꿈, 단락)이 그대로 유지됩니다.

## 자주 묻는 질문

### 사진이 없어도 글을 생성할 수 있나요?
최소 1장의 사진이 필요합니다. 사진은 AI가 글의 맥락을 파악하는 데 활용됩니다.

### 생성된 글의 길이는 어느 정도인가요?
500~1500자 사이로 생성됩니다. 사진 수가 많을수록 글이 길어지는 경향이 있습니다.

### 같은 사진/메모로 다시 생성하면 같은 결과가 나오나요?
아니요. AI가 매번 다르게 작성하므로 결과가 달라집니다. 마음에 드는 결과가 나올 때까지 재생성해볼 수 있습니다.

### 수정 요청을 여러 번 할 수 있나요?
네. 이전 대화를 기억하므로 "좀 더 짧게", "이모지 추가해줘" 등 점진적으로 수정할 수 있습니다.

### 서버를 껐다 켜면 수정이 가능한가요?
아닙니다. 세션 데이터가 서버 메모리에 저장되므로, 서버 재시작 시 기존 세션이 소실됩니다. 수정이 필요하면 서버가 실행 중인 동안 완료해 주세요.

### 어떤 AI 모델을 사용하나요?
Anthropic의 Claude Sonnet (`claude-sonnet-4-6`)을 사용합니다. `application.yml`에서 모델을 변경할 수 있습니다.

## 문제 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| 실행 시 "Gradle requires JVM 17" | Java 버전이 17 미만 | `JAVA_HOME=$(/usr/libexec/java_home -v 17) ./gradlew bootRun` |
| "401 Unauthorized" 에러 | API 키 미설정 또는 잘못됨 | `.env` 파일의 `CLAUDE_API_KEY` 값 확인 |
| 글 생성이 매우 오래 걸림 | 사진 수가 많거나 용량이 큼 | 사진 수를 줄이거나 미리 리사이즈 |
| "사진을 1장 이상 업로드해 주세요" | 파일이 선택되지 않음 | 지원 포맷(JPEG/PNG/GIF/WebP) 확인 |
| 생성된 글이 깨져 보임 | AI 응답 형식 이탈 | 재생성 시도 |
